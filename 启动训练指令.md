# SFA4D 启动指令文档

## 概述

本文档包含SFA4D项目的常用启动指令，包括训练、推理、测试等操作的完整命令。
注：我们训练读取0，1，2，4索引的点云，即将第五维信噪比强度P作为强度。
维度读取逻辑:
8D数据: 通过 process_8d_lidar_data_scheme_B() 读取维度 [0,1,2,4] (P映射为intensity)
即：8D数据 [x,y,z,D,P,R,A,E] 
→ 读取 [0,1,2,4] 维度 
→ P映射为intensity 
→ 输出4D [x,y,z,intensity]
## 环境要求

- Python 3.8+
- PyTorch 2.0.0+cu118
- CUDA 11.8+
- conda环境：sfa3d

## 激活环境

```bash
source activate sfa3d
```

## 训练指令

### 测试训练（3轮测试）

```bash
source activate sfa3d && python sfa/train.py --num_epochs 3 --saved_fn sfa4d_test_3epoch --checkpoint_freq 1 --print_freq 10 --batch_size 4 --dataset-dir ./DRadDataset --root-dir ./ --gpu_idx 0
```

**参数说明：**
- `--num_epochs 3`: 训练3个epoch
- `--saved_fn sfa4d_test_3epoch`: 模型保存文件名前缀
- `--checkpoint_freq 1`: 每轮保存模型
- `--print_freq 10`: 每10个batch打印一次日志
- `--batch_size 4`: 批次大小为4
- `--dataset-dir ./DRadDataset`: 数据集相对路径
- `--root-dir ./`: 项目根目录
- `--gpu_idx 0`: 使用第一个GPU

### 完整训练（300轮）

```bash
source activate sfa3d && python sfa/train.py --num_epochs 300 --saved_fn sfa4d_full --checkpoint_freq 1 --print_freq 50 --batch_size 16 --dataset-dir ./DRadDataset --root-dir ./ --gpu_idx 0
```

### 恢复训练

```bash
source activate sfa3d && python sfa/train.py --num_epochs 300 --saved_fn sfa4d_resume --checkpoint_freq 1 --print_freq 50 --batch_size 16 --dataset-dir ./DRadDataset --root-dir ./ --gpu_idx 0 --resume_path ./checkpoints/your_model/Model_your_model_epoch_X.pth
```

## 常见问题

### 1. 路径问题
确保使用相对路径 `./DRadDataset` 而不是绝对路径。

### 2. GPU内存不足
减少 `batch_size` 参数，从8改为4或2。

### 3. 多GPU训练
使用 `--multiprocessing-distributed` 参数。

### 4. 恢复训练路径
检查 `--resume_path` 参数指向的模型文件是否存在。
