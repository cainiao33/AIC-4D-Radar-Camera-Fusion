# SFA4D 教学案例：网络架构详解

## 1. 文档定位

本文面向课堂教学与项目复盘，系统讲解 SFA4D 在 2025 年 AIC 算法竞赛中的最终网络方案。该方案获得**国赛二等奖**，全国排名 **10/85**，是一个兼具竞赛成绩、工程完整性与教学价值的 3D 目标检测案例。

---

## 2. 总体架构

SFA4D 的最终网络采用 **8D毫米波雷达点云 → 4D兼容映射 → BEV投影 → ResNet-18 + FPN + KFPN → 多任务检测头 → 跨类别NMS** 的技术路线。

```text
8D Radar Point Cloud
[x, y, z, Doppler, P, Range, Azimuth, Elevation]
        ↓
8D→4D 智能映射 [x, y, z, P]
        ↓
BEV 三通道投影 [Intensity, Height, Density]
        ↓
ResNet-18 Backbone
        ↓
FPN 多尺度特征金字塔
        ↓
KFPN Softmax 自适应加权融合
        ↓
5个检测头：hm_cen / cen_offset / z_coor / dim / direction
        ↓
3D检测框 + 类别 + 置信度
```

---

## 3. 输入层：8D毫米波雷达数据建模

### 3.1 原始数据格式

竞赛数据来自 DRadDataset，每个点包含 8 个维度：

| 维度 | 含义 | 作用 |
|------|------|------|
| x | 前向距离 | 空间定位 |
| y | 横向距离 | 空间定位 |
| z | 高度 | 3D定位 |
| Doppler | 多普勒速度 | 运动信息 |
| P | 信噪比强度 | 回波质量，最终选用 |
| Range | 极坐标距离 | 几何冗余信息 |
| Azimuth | 方位角 | 几何角度信息 |
| Elevation | 俯仰角 | 几何角度信息 |

### 3.2 关键设计：8D→4D智能映射

一般的激光雷达3D检测算法主要面向 `[x, y, z, intensity]` 这类4维点云设计，无法直接处理较新的4D/高维毫米波雷达数据。SFA4D 没有简单堆叠更多通道，而是从雷达信号物理特性出发，选择**最有判别力且最适合BEV检测建模**的维度：

```text
输入: [x, y, z, D, P, R, A, E]
选择: [0, 1, 2, 4]
输出: [x, y, z, P]
```

其中 `P` 被映射为 intensity 通道。

### 3.3 教学价值

该设计非常适合课堂讨论：

- **不是维度越多越好**：多余维度可能增加噪声和训练难度。
- **算法设计要尊重数据物理意义**：P 反映雷达回波信号质量，与目标存在更稳定关联。
- **工程落地性很重要**：通过 4D 表征重构，让较新的毫米波雷达数据能够进入成熟的BEV检测范式，同时保留雷达信号的关键判别信息。

## 4. BEV投影层

### 4.1 空间范围

| 参数 | 范围 |
|------|------|
| X轴 | 0m ~ 50m |
| Y轴 | -25m ~ 25m |
| Z轴 | -2.73m ~ 1.27m |
| BEV尺寸 | 608 × 608 |
| 输出特征图 | 152 × 152 |

### 4.2 三通道BEV特征

SFA4D 将点云转为三通道 BEV 图像：

| 通道 | 含义 | 教学解释 |
|------|------|----------|
| Intensity Map | P强度最大值 | 目标回波质量 |
| Height Map | 最大高度 | 目标垂直结构 |
| Density Map | 点密度 | 局部点云聚集程度 |

### 4.3 为什么使用BEV？

- **降低复杂度**：3D点云转为2D图像，可直接使用CNN。
- **保持空间结构**：俯视图天然适合车辆、骑行者、卡车等地面目标检测。
- **适配实时性**：比体素网络和点云网络更快。

---

## 5. Backbone：ResNet-18

### 5.1 选择ResNet-18的原因

最终模型选择 ResNet-18 作为骨干网络，而不是 ResNet-34/50，核心原因是竞赛需要在**精度、速度、模型大小**之间平衡。

| Backbone | 特点 | 适用性 |
|----------|------|--------|
| ResNet-18 | 快、轻量、易训练 | 最终采用 |
| ResNet-34 | 精度略高，速度下降 | 备选 |
| ResNet-50 | 表达能力强，部署成本高 | 不适合实时竞赛场景 |

### 5.2 残差结构教学重点

ResNet 的核心是残差连接：

```text
Output = F(x) + x
```

它解决了深层网络训练中的梯度退化问题，使模型能够稳定提取毫米波雷达 BEV 图像中的多层次特征。

---

## 6. Neck：FPN多尺度特征金字塔

### 6.1 FPN作用

不同目标尺度差异明显：

- Car：目标大，特征稳定。
- Cyclist：目标小，点云稀疏。
- Truck：尺寸大但形态变化多。

FPN 通过高层语义与低层细节融合，提高不同尺度目标的检测能力。

### 6.2 特征融合流程

```text
Layer4 高语义特征
      ↑ 上采样
Concat Layer3
      ↑ 上采样
Concat Layer2
      ↑ 上采样
Concat Layer1
```

课堂讲解可强调：

- 高层特征负责“是什么”。
- 低层特征负责“在哪里”。
- FPN 把二者结合起来。

---

## 7. 核心创新：KFPN自适应加权融合

### 7.1 KFPN思想

传统 FPN 往往采用直接相加或拼接，默认每层特征同等重要。SFA4D 引入 KFPN，通过 Softmax 在不同金字塔层之间学习权重。

核心逻辑：

```text
FPN输出: f1, f2, f3
        ↓
Softmax计算层间权重
        ↓
加权求和得到最终预测特征
```

### 7.2 代码级逻辑

```python
outs = torch.cat([out.unsqueeze(-1) for out in outs], dim=-1)
softmax_outs = F.softmax(outs, dim=-1)
ret_outs = (outs * softmax_outs).sum(dim=-1)
```

### 7.3 教学解释

KFPN 可以理解为模型在回答：

> 对当前像素位置和当前类别，应该更相信哪一层特征？

这比固定融合更灵活，尤其对小目标和尺度变化明显的场景更有效。

---

## 8. 检测头设计

SFA4D 采用 Anchor-Free 多任务检测头，直接预测目标中心点和3D属性。

| Head | 输出维度 | 预测内容 |
|------|----------|----------|
| hm_cen | 3 | Car / Cyclist / Truck 中心点热力图 |
| cen_offset | 2 | 中心点亚像素偏移 |
| z_coor | 1 | 目标Z坐标 |
| dim | 3 | 目标长宽高 |
| direction | 2 | 朝向角 sin/cos 编码 |

### 8.1 为什么不用Anchor？

Anchor-Free 的优势：

- 减少锚框尺寸、比例、方向等超参数。
- 更适合 BEV 中心点检测。
- 推理更快，符合实时竞赛要求。

---

## 9. 后处理：跨类别NMS

### 9.1 问题背景

常规 NMS 只在同类别内抑制重复框，但毫米波雷达数据稀疏、噪声大，容易出现一个目标被识别为多个类别的问题。

### 9.2 最终策略

| 参数 | 最终值 | 作用 |
|------|--------|------|
| peak_thresh | 0.25 | 降低中心点门槛，提高召回 |
| nms_thresh | 0.2 | 更严格过滤重叠框 |
| inter_class_nms | True | 跨类别去重 |

该策略是最终竞赛成绩提升的重要工程优化点。

---

## 10. 网络方案总结

SFA4D 最终网络的优秀之处在于：

- **数据层聪明**：8D→4D不是粗暴降维，而是基于物理含义选择 P 维度。
- **架构层稳健**：ResNet-18 + FPN 是成熟可靠的实时检测组合。
- **创新层有效**：KFPN让多尺度融合具备自适应能力。
- **输出层清晰**：Anchor-Free多任务头直接回归3D检测结果。
- **后处理层务实**：跨类别NMS针对竞赛误检问题精准优化。
