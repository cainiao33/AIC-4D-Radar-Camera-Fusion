## 2. 摘要

SFA4D 是一个面向 8D 毫米波雷达点云数据的实时 3D 目标检测系统。项目针对一般激光雷达3D检测算法难以直接处理较新的4D/高维毫米波雷达数据、雷达点云稀疏且噪声强等问题，提出了以**8D→4D智能维度映射**为核心的数据建模方案，并在此基础上构建了 **ResNet-18 + FPN + KFPN** 的实时检测网络。

系统采用 Anchor-Free 检测范式，通过多任务检测头同时预测目标中心点、中心偏移、Z坐标、3D尺寸和方向角。后处理阶段引入跨类别NMS策略，有效缓解毫米波雷达检测中常见的跨类别重复框问题。

最终方案在 DRadDataset 验证集上实现 **75% mAP@0.5**、**110.73 FPS** 的实时推理性能，并完成 ONNX 导出与模型量化，量化后模型大小从 48.66MB 降至 13MB。该方案在 2025 年 AIC 算法竞赛中获得**国赛二等奖**，全国排名 **10/85**。

## 3. 问题背景

### 3.1 自动驾驶感知中的毫米波雷达价值

毫米波雷达具备以下优势：

1. **全天候感知**：不受雨、雪、雾、强光和夜间环境影响。
2. **成本较低**：相较 LiDAR 更适合规模化部署。
3. **速度信息天然可得**：可直接提供多普勒速度信息。
4. **工程可靠性高**：在辅助驾驶和自动驾驶系统中已有广泛应用。

### 3.2 技术挑战

毫米波雷达也存在明显挑战：

| 挑战 | 具体表现 |
|------|----------|
| 点云稀疏 | 目标点数少，小目标更难识别 |
| 噪声较强 | 易产生误检和重复检测 |
| 高度信息弱 | 3D定位精度低于LiDAR |
| 数据维度复杂 | 原始8D信息难以直接输入传统4D框架 |
| 实时要求高 | 自动驾驶场景需要高FPS低延迟 |

本项目的核心任务就是：**在不牺牲实时性的前提下，将8D毫米波雷达数据有效转化为可学习、可检测、可部署的3D目标检测系统。**

## 4. 技术路线

### 4.1 总体流程

```text
8D毫米波雷达点云
        ↓
维度分析与智能映射
        ↓
4D点云 [x, y, z, P]
        ↓
BEV三通道投影
        ↓
ResNet-18特征提取
        ↓
FPN多尺度融合
        ↓
KFPN自适应加权
        ↓
Anchor-Free多任务检测头
        ↓
跨类别NMS后处理
        ↓
KITTI格式3D检测结果
```

### 4.2 关键技术选择

| 模块 | 方案 | 选择原因 |
|------|------|----------|
| 数据映射 | `[x,y,z,D,P,R,A,E] → [x,y,z,P]` | 保留物理意义最强的P维度 |
| 空间表达 | BEV投影 | 兼顾结构表达与实时性 |
| Backbone | ResNet-18 | 速度快、训练稳定、部署友好 |
| Neck | FPN | 多尺度目标检测能力强 |
| 融合模块 | KFPN | 自适应选择不同尺度特征 |
| 检测范式 | Anchor-Free | 减少锚框调参，提升推理效率 |
| 后处理 | 跨类别NMS | 抑制跨类重复检测 |

## 5. 核心创新

### 5.1 创新一：8D→4D智能维度映射

#### 问题

一般激光雷达3D检测算法通常面向 `[x,y,z,intensity]` 四维点云设计，而本竞赛数据是包含多普勒、信噪比强度、距离、角度等信息的 8D 毫米波雷达点云。如果直接扩展输入通道，会引入额外计算成本与噪声；如果简单丢弃维度，则可能损失关键信息。

#### 方法

经过数据分析与实验对比，最终选择 `[0,1,2,4]` 四个维度：

```text
原始8D: [x, y, z, Doppler, P, Range, Azimuth, Elevation]
最终4D: [x, y, z, P]
```

其中 `P` 表示信噪比强度，被映射为传统点云中的 intensity 通道。

#### 优势

- **物理意义明确**：P反映雷达回波信号质量。
- **目标区分能力强**：相比Doppler、Range等维度更稳定。
- **工程落地性好**：将高维雷达信号转化为适合BEV检测的紧凑表达，兼顾检测精度、计算效率和系统可部署性。
- **计算代价低**：避免设计复杂多分支网络。

---

### 5.2 创新二：KFPN自适应多尺度特征融合

#### 问题

毫米波雷达目标尺度变化明显，Car、Cyclist、Truck 在点云密度和BEV尺度上差异较大。传统FPN简单融合多层特征，难以动态判断不同层的重要性。

#### 方法

在FPN基础上引入 KFPN（Keypoint Feature Pyramid Network），对不同尺度预测特征进行 Softmax 加权融合：

```text
多尺度特征 f1, f2, f3
        ↓
Softmax生成权重 w1, w2, w3
        ↓
融合特征 F = w1*f1 + w2*f2 + w3*f3
```

#### 效果

KFPN 相比简单拼接与加权平均取得更优检测表现，尤其改善 Cyclist 等小目标类别。

---

### 5.3 创新三：跨类别NMS后处理优化

#### 问题

毫米波雷达点云稀疏且噪声大，同一真实目标可能被多个类别同时检测出来。传统NMS只在同类别内抑制，无法解决跨类别重复框问题。

#### 方法

最终采用：

| 参数 | 数值 |
|------|------|
| peak_thresh | 0.25 |
| nms_thresh | 0.2 |
| 跨类别NMS | 开启 |

#### 效果

该策略在保持较高召回率的同时，有效减少重复检测，是最终成绩提升的重要工程环节。

## 6. 网络结构

### 6.1 输入与BEV表示

输入点云经过裁剪和投影，生成 608×608 的三通道 BEV 图像：

| 通道 | 内容 |
|------|------|
| 强度通道 | P维度映射值 |
| 高度通道 | 网格内最大高度 |
| 密度通道 | 网格点云数量 |

### 6.2 主干网络

采用 ResNet-18 提取多层语义特征。该网络具备训练稳定、推理速度快、参数规模适中的优势。

### 6.3 FPN与KFPN

FPN负责多尺度特征融合，KFPN负责自适应加权不同尺度输出。最终输出分辨率为 152×152。

### 6.4 多任务检测头

| 分支 | 任务 |
|------|------|
| hm_cen | 类别中心点热力图 |
| cen_offset | 中心点偏移 |
| z_coor | 垂直坐标 |
| dim | 3D尺寸 |
| direction | 朝向角 |

该结构实现了端到端 7-DOF 目标检测。

## 7. 训练策略

### 7.1 训练配置

| 参数 | 配置 |
|------|------|
| 总训练轮数 | 300 epochs |
| 最佳模型 | Epoch 163 |
| 优化器 | Adam |
| 输入尺寸 | 608×608 |
| 检测类别 | Car / Cyclist / Truck |
| 数据增强 | 翻转、旋转、缩放 |

### 7.2 损失函数

总损失由多个任务损失加权组成：

```text
L_total = L_heatmap + L_offset + L_z + L_dim + L_direction
```

| 损失 | 用途 |
|------|------|
| Focal Loss | 解决热力图正负样本不平衡 |
| L1 Loss | 中心偏移与方向回归 |
| Balanced L1 Loss | 尺寸与Z坐标回归 |

## 9. 消融实验结论

### 9.1 维度映射

P维度作为 intensity 明显优于 Doppler、Range、RCS 等候选维度，是精度提升的关键来源之一。

### 9.2 特征融合

KFPN 相比简单拼接约提升 3% mAP，相比普通加权平均约提升 1% mAP。

### 9.3 后处理策略

跨类别NMS减少了毫米波雷达场景中的重复检测，并提升最终提交稳定性。

## 11. 项目不足与后续改进

### 11.1 当前不足

1. **小目标检测仍有难度**：Cyclist点云稀疏，远距离表现下降。
2. **8D信息利用不充分**：Doppler等速度信息尚未完全发挥价值。
3. **ONNX推理速度待优化**：后续可引入TensorRT或图优化。
4. **跨类别NMS可能误删近邻目标**：复杂密集场景仍需进一步优化。

### 11.2 后续方向

| 方向 | 改进方案 |
|------|----------|
| 多模态融合 | 融合相机或LiDAR信息 |
| 时序建模 | 利用多帧雷达点云与Doppler速度 |
| 轻量化 | 引入MobileNet、剪枝、蒸馏 |
| 部署加速 | TensorRT、INT8量化、边缘端优化 |
| 后处理优化 | 距离自适应NMS、类别自适应阈值 |
