# SFA3D 点云维度修改说明

## 修改概述

本项目是点云数据读取逻辑，使其使用维度 **0, 1, 2, 4** 而不是原来的 **0, 1, 2, 3**。

## 修改目的

- **原始格式**: 使用点云的前4个维度 [X, Y, Z, Intensity]
- **修改后格式**: 使用点云的维度 [X, Y, Z, 第5个维度]，跳过第4个维度(Intensity)

## 修改的文件

### 1. `sfa/data_process/kitti_dataset.py`

**修改位置**: `get_lidar()` 函数 (第160-182行)

**修改内容**:
- 原来直接读取为 `(-1, 4)` 形状
- 现在先尝试读取为 `(-1, 5)` 形状，然后选择维度 `[0, 1, 2, 4]`
- 如果数据不是5维，会自动回退到原始4维格式并显示警告

```python
# 修改前
return np.fromfile(lidar_file, dtype=np.float32).reshape(-1, 4)

# 修改后
lidar_data = np.fromfile(lidar_file, dtype=np.float32)
if len(lidar_data) % 5 == 0:
    lidar_data = lidar_data.reshape(-1, 5)
    selected_data = lidar_data[:, [0, 1, 2, 4]]  # 选择维度0,1,2,4
    return selected_data
else:
    # 回退到4维格式
    return lidar_data.reshape(-1, 4)
```

### 2. `sfa/data_process/demo_dataset.py`

**修改位置**: `get_lidar()` 函数 (第96-118行)

**修改内容**: 与 `kitti_dataset.py` 中的修改完全一致

### 3. `sfa/data_process/kitti_bev_utils.py`

**状态**: **无需修改**

**原因**: `makeBEVMap()` 函数期望输入4维数据 `[x, y, z, intensity]`，我们的修改后数据仍然是4维，只是第4个维度的含义从 intensity 变为第5个原始维度。

## 数据兼容性

### 5维数据 (推荐)
如果您的点云数据有5个维度，代码会：
1. 自动检测数据维度
2. 选择维度 0, 1, 2, 4
3. 返回4维数据供后续处理

### 4维数据 (兼容模式)
如果您的点云数据只有4个维度，代码会：
1. 检测到数据不是5维
2. 显示警告信息
3. 使用原始4维数据

## 使用说明

### 数据准备
确保您的点云数据文件 (`.bin`) 包含5个维度：
- 维度 0: X 坐标
- 维度 1: Y 坐标  
- 维度 2: Z 坐标
- 维度 3: Intensity (将被跳过)
- 维度 4: 您想要使用的第5个维度

### 训练/推理
使用方式与原始项目完全相同：

```bash
# 训练
python sfa/train.py --config_path sfa/config/train_config.py

# 测试
python sfa/test.py --config_path sfa/config/test_config.py
```

## 注意事项

1. **数据格式**: 确保您的点云数据确实有5个维度
2. **向后兼容**: 如果数据只有4个维度，代码会自动回退到原始格式
3. **性能**: 修改不会影响模型的训练和推理性能
4. **BEV映射**: `makeBEVMap` 函数会将第4个维度(原第5个维度)用作强度图
